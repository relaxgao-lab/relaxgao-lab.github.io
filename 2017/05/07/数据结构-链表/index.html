概念了解

链表（Linked List）是一种常用的数据结构，常用于需要频繁的插入、删除操作的场景。链表的元素不是连续存储的，而是通过指针链接起来的，每个元素包含一个指向下一个元素的指针。链表包括单向链表、双向链表和循环链表等。

单向链表（Singly Linked List）：每个节点只包含一个指向下一个节点的指针，最后一个节点指向空。

双向链表（Doubly Linked List）：每个节点包含一个指向下一个节点和一个指向前一个节点的指针，第一个节点前一个节点和最后一个节点后一个节点都指向空。

循环链表（Circular Linked List）：和单向链表或双向链表类似，但最后一个节点指向第一个节点形成循环。

链表与数组相比，链表的插入、删除操作更加高效，但是访问元素的时间复杂度是O(n)，而数组的时间复杂度是O(1)。在实际应用中，需要根据不同的需求选择合适的数据结构。

  • Handler 消息队列的实现：在 Android 中，Handler 是实现线程间通信的一种机制，它使用了消息队列来存储线程发送的消息。消息队列是使用链表来实现的，每个消息都被封装成一个 Message 对象，使用链表链接起来，形成一个消息队列。

    示例代码实现了一个简单的 Handler 类，其中包括了一个 MessageQueue 类和一个 Message 类，用来实现消息队列和消息的封装。Handler 的 sendMessage 方法会将消息添加到消息队列中，dispatchMessage 方法会将消息从队列中取出并进行处理。在循环中调用 MessageQueue 的 next 方法，可以获取队列中的下一个消息进行处理。当队列为空时，使用 wait 方法等待新的消息加入队列。每个 Message 对象中包含了消息类型、消息内容、消息目标 Handler 等信息，同时也可以包含一个 Runnable 类型的 callback，用来执行任务。
