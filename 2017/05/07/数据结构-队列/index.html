概念了解

队列是一种常见的数据结构，类似于现实生活中的排队场景。队列的特点是先进先出（First-In-First-Out，FIFO），即先加入队列的元素会先被取出。

队列在android里应用

队列在 Android 中有很多应用场景，以下是一些常见的场景：

  1. 消息队列

    Android 中的 Handler 类就是基于消息队列实现的，可以通过 Handler 向消息队列中发送消息，然后由 Looper 从消息队列中取出消息并交给相应的 Handler 进行处理。在 Android 中，消息队列的应用非常广泛，例如可以用来实现异步任务、延迟执行、任务调度等。

  2. 任务队列

    在 Android 中，可以通过线程池来管理任务队列，将一些需要执行的任务添加到任务队列中，线程池会自动从任务队列中取出任务并执行。这样可以避免创建过多的线程导致系统资源浪费和性能下降。

  3. 消息通知队列

    Android 中的通知栏和 Toast 通知都是通过消息队列来实现的。当用户收到通知时，系统会将通知消息添加到消息队列中，然后在合适的时机将通知显示在通知栏或者屏幕上。

  4. 事件队列

    在 Android 中，当用户触摸屏幕、点击按钮等操作时，系统会将相应的事件添加到事件队列中，然后由主线程从事件队列中取出事件并进行处理。这样可以保证用户交互的响应速度和稳定性。

  5. 图像加载队列

    在 Android 中，可以通过使用图片加载框架来实现图片加载队列，将需要加载的图片添加到队列中，图片加载框架会自动从队列中取出任务并进行图片加载，同时可以通过缓存机制和异步加载等方式提高图片加载的性能和效率。
