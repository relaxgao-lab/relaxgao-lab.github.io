定义

超文本传输协议（HTTP）是一个用于传输超媒体文档（例如 HTML）的应用层协议。它是为 Web 浏览器与 Web 服务器之间的通信而设计的，但也可以用于其他目的。HTTP 遵循经典的客户端—服务端模型，客户端打开一个连接以发出请求，然后等待直到收到服务器端响应。HTTP 是无状态协议，这意味着服务器不会在两个请求之间保留任何数据（状态）。

用于浏览器

⽤户输⼊地址后回⻋或点击链接 -> 浏览器拼装 HTTP 报⽂并发送请求给服务器 -> 服 务器处理请求后发送响应报⽂给浏览器 -> 浏览器解析响应报⽂并使⽤渲染引擎显示 到界⾯

用于app

⽤户点击或界⾯⾃动触发联⽹需求 -> Android 代码调⽤拼装 HTTP 报⽂并发送请求 到服务器 -> 服务器处理请求后发送响应报⽂给⼿机 -> Android 代码处理响应报⽂并作出相应处理。

http请求和相应

请求

HTTP 请求的内容通称为”资源”。”资源“这一概念非常宽泛，它可以是一份文档，一张图片，或所有其他你能够想到的格式。每个资源都由一个 (URI) 来进行标识。

请求报文：

字段说明：

  • ET /path/to/resource HTTP/1.1：请求行，包括请求方法（ET）、请求路径（/path/to/resource）和 HTTP 版本（HTTP/1.1）。
  • Host: example.com：请求头，指定请求的主机名和端口号。
  • User-Agent: Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/89.0.4389.82 Safari/537.36：请求头，指定客户端浏览器的信息，包括浏览器类型、操作系统和版本号等。
  • Accept: text/html,application/xhtml+xml,application/xml;q=0.9,image/webp,image/apng,*/*;q=0.8,application/signed-exchange;v=b3;q=0.9：请求头，指定客户端可以接受的响应类型。
  • Accept-Encoding: gzip, deflate, br：请求头，指定客户端可以接受的压缩算法。
  • Accept-Language: en-US,en;q=0.9：请求头，指定客户端可以接受的语言。
  • Connection: keep-alive：请求头，指定客户端与服务器的连接是否保持活动状态。
响应报文

字段说明：

  • HTTP/1.1 200 OK：响应行，包括HTTP版本号（1.1）、状态码（200）和状态消息（OK）。
  • Date: Wed, 12 May 2023 08:00:00 GMT：表示响应的日期和时间，格式为 RFC 1123。
  • Server: Apache/2.4.6 (CentOS)：服务器信息，表示响应的是 Apache 2.4.6 版本。
  • Last-Modified: Tue, 11 May 2023 10:00:00 GMT：表示资源的最后修改时间，格式为 RFC 1123。
  • ETag: "123456-abcdef"：表示资源的实体标识符，可以用于缓存控制。
  • Content-Length: 1234：表示响应体的长度（以字节为单位）。
  • Content-Type: text/html; charset=UTF-8：表示响应体的 MIME 类型和字符编码。
  • <!DOCTYPE html>...：表示响应体，即实际的文档内容。

常见的相应状态码：

  • 200 OK：请求成功，服务器已成功返回请求所需信息。通常用于 GET 和 POST 请求。

  • 201 Created：请求成功，服务器已成功创建资源。通常用于 POST 请求。

  • 204 No Content：请求成功，服务器已成功处理请求，但不需要返回任何实体内容。通常用于 DELETE 请求。

  • 301 Moved Permanently：请求的资源已经被永久移动到新的 URL，所有对该资源的请求都应该使用新的 URL。301状态码的响应头里通常包括一个Location字段，用于指定新的URL。例如：

  • 302 Found：请求的资源临时被移动到新的 URL，所有对该资源的请求都应该使用新的 URL。与 301 状态码不同，302 状态码表示请求的资源只是暂时被移动。302状态码的响应头里也通常包括一个Location字段，用于指定新的URL。例如：

  • HTTP/1.1 302 Found
Location: https://www.example.com/new-url

  • 304 Not Modified：请求的资源未修改，服务器不需要返回任何实体内容，客户端可以使用本地缓存的内容。通常用于 GET 请求。

  • 400 Bad Request：请求无效，服务器无法理解请求的格式，语法或参数。

  • 401 Unauthorized：请求需要身份验证，客户端需要提供有效的身份验证信息。

  • 403 Forbidden：服务器已经理解请求，但是拒绝执行它，因为客户端没有访问权限。

  • 404 Not Found：服务器无法找到请求的资源，通常用于 GET 请求。

  • 500 Internal Server Error：服务器内部错误，无法完成请求。
